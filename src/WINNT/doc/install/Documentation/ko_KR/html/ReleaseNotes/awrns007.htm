AFS 제어 센터 릴리스 노트

소개

이 절에는 Windows용 AFS 버전 3.6의 AFS 제어 센터 구성요소에 대한 중요한 정보가 들어 있습니다. AFS 제어 센터에는 AFS 시스템 관리자의 AFS 셀 관리를 돕기 위한 두 개의 강력한 그래픽 사용자 인터페이스(GUI), 즉 AFS 서버 관리 프로그램과 AFS 계정 관리 프로그램이 있습니다.

AFS 제어 센터 설치 요구조건

사용자 시스템은 Windows용 AFS 제어 센터 구성요소를 설치하기 위해 다음과 같은 하드웨어, 소프트웨어 및 관리 특권 요구조건을 충족시켜야 합니다.

하드웨어

AFS 제어 센터를 설치하는 하드웨어 요구조건은 다음과 같습니다.

  • 80486/66 MHz(이상) 마이크로프로세서 또는 펜티엄 기반 마이크로프로세서를 사용하는 PC
  • 최소 16MB의 RAM
  • 최소 4MB의 사용가능한 디스크 공간을 가진 하드 디스크

소프트웨어

AFS 제어 센터를 설치하는 소프트웨어 요구조건은 다음과 같습니다.

  • 서비스 팩 4 또는 5가 있는 Microsoft Windows NT 4.0

관리 특권 요구조건

AFS 제어 센터를 설치하려면 Windows 시스템에서 국지 관리자 그룹의 구성원이어야 합니다.

제품 주의사항

이 절은 Windows용 AFS 버전 3.6의 AFS 제어 센터 구성요소에 대한 중요한 일반 정보를 강조합니다.

AFS 제어 센터는 경로가 ANSI 문자만을 포함하는 디렉토리에 설치되어야 합니다

Windows용 AFS 소프트웨어는 경로에 ANSI 문자만이 들어 있는 디렉토리에만 설치될 수 있습니다(예를 들어, C:\Program Files).

임시 파일

Windows용 AFS 제어 센터는 경우에 따라 임시 파일을 작성합니다. 그러한 파일이 작성되는 곳을 제어하려면, 선호하는 임시 디렉토리의 전체 경로에 환경 변수(TMP 또는 TEMP)를 설정하십시오. 임시 디렉토리를 지정하지 않는 경우, 임시 파일은 파일을 작성한 프로세스의 현재 작업 디렉토리에 작성됩니다.

AFS 제어 센터에 대한 자국어 지원

AFS는 양방향 스크립트(Hebrew, Arabic 등)에 대한 지원을 포함하여, AFS 제어 센터 그래픽 사용자 인터페이스(GUI) 도구 및 문서에 대한 자국어 지원을 제공합니다. 설치된 언어 문자열은 제어판의 국가별 설정 상자 대화 상자에 지정한 대로 사용자 기계의 시스템 기본 로케일에 의해 결정됩니다.
주:기계의 시스템 기본 로케일은 기계에서의 사용자 로케일(사용자가 지정)과 다를 수 있습니다. 그러나, Windows용 AFS가 설치된 언어 문자열은 항상 시스템 기본 로케일에 의해 결정됩니다.

현재 로케일에 대한 언어 문자열이 없는 경우, 영어 문자열이 기본값으로 설치됩니다.

kpwvalid Password Strength Checker는 Windows 시스템에서 사용되지 않습니다

AFS 사용자 암호가 Windows 시스템에서 변경되면, kpwvalid 프로그램은 새 암호를 확인하기 위해 사용되지 않습니다 (UNIX 시스템에서, kpwvalid 프로그램이 kpasswd 프로그램과 동일한 디렉토리에 존재하는 경우, kpwvalid 프로그램은 모든 새 사용자 암호를 확인합니다).

25시간 이후에 제어 센터 응용 프로그램 만기 시 획득되는 토큰

AFS Server Manager 또는 AFS Account Manager 중 하나를 사용하려면, 사용자는 AFS 토큰을 획득해야 합니다. 사용자용으로 지정된 최대 티켓 수명에 관계없이, 응용프로그램 중 하나에 획득된 토큰의 수명은 항상 25시간 25분입니다.

서버 관리 프로그램은 볼륨 위치 데이터베이스(VLDB)에서 참조하는 모든 서버 항목을 표시합니다

AFS 서버 관리 프로그램은 볼륨 위치 데이터베이스(VLDB)에서 모든 서버 항목에 대한 서버 아이콘을 표시합니다. AFS 서버의 사용이 중지되었으나 관련 VLDB 항목이 제거되지 않은 경우, 서버 관리 프로그램은 서버에 대한 아이콘을 계속 표시합니다.

AFS 버전 3.5 이상을 실행하는 데이터베이스 서버에 있는 VLDB에서 쓸모없는 AFS 파일 서버 항목을 제거하려면, 다음 명령을 발행하십시오. 

vos changeaddr <ip_address>
-remove

여기서 ip_address는 사용이 중지된 서버 기계의 인터넷 프로토콜(IP) 주소입니다.

버전 3.5 이전 버전을 실행하는 AFS 데이터베이스 서버에서는, VLDB에서 쓸모없는 서버 항목을 제거하는 명령이 현재 없습니다. 그러나, AFS 서버 관리 프로그램이 사용이 중지된 서버를 표시하지 못하게 하기 위해, Windows NT 레지스트리를 수정하여 AFS 서버 관리 프로그램이 지정된 패턴의 IP 주소를 가진 모든 기계를 무시할 수 있습니다.

AFS 서버 관리 프로그램이 지정된 패턴의 IP 주소를 가진 서버 기계를 표시하지 못하게 하려면, 다음 단계를 수행하여 Windows NT 레지스트리를 수정하십시오.

  1. 시작 메뉴에서, 실행을 선택하십시오. 열기 상자에서, regedit를 입력한 후 확인을 선택하십시오. Windows NT 레지스트리 편집기가 열립니다.
  2. \\HKEY_LOCAL_MACHINE\Software\TransarcCorporation\AFS 제어 센터 키의 오른쪽을 누르십시오. 열린 컨텍스트 메뉴에서, 신규를 선택한 후 DWORD 값을 선택하십시오. 새 값 항목이 키에 추가됩니다.
  3. 새 값으로서 IgnoreBadAddrs를 입력하여 새 값 항목의 이름을 지정하십시오.
  4. 새 IgnoreBadAddrs 항목을 두 번 누르십시오. 편집 DWORD 값 대화 상자가 열립니다.
  5. 값 데이터 상자에서, 무시할 IP 주소나 IP 주소의 범위를 16진수 표기로 지정하십시오. 예를 들어, AFS 서버 관리 프로그램이 패턴 10.0.0.*와 일치하는 IP 주소로 VLDB에 나열된 모든 기계를 무시하도록 지시하려면, 0A0000FF를 값 데이터 상자에 입력하십시오(16진수 값 FF는 와일드 카드를 나타냅니다).
  6. 확인을 눌러 Edit DWORD 값 상자를 닫고 새 레지스트리 항목을 저장하십시오.
  7. 레지스트리 편집기를 닫으십시오.

주:이 기능을 이용하려면, 지정 범위에서 쉽게 인식할 수 있는 미사용 IP 주소로 VLDB에서 사용이 중지된 서버의 IP 주소를 변경하도록 권장합니다(예를 들어, 10.0.0.1, 10.0.0.2, 10.0.0.3 등). 그런 다음 이전 단계에서 기술된 대로 IgnoreBadAddrs 레지스트리를 추가하여 AFS 서버 관리 프로그램이 지정된 범위 내에서 모든 IP 주소를 무시합니다.

제한사항 및 제약사항

Windows용 AFS, 버전 3.6의 AFS 제어 센터 구성요소에 대해 현재 알려진 제한사항 및 제약사항은 없습니다.
