AFS 서버 릴리스 노트

소개

이 절에는 Windows용 AFS 버전 3.6의 AFS 서버 구성요소에 대한 중요한 정보가 들어 있습니다. AFS 서버는 Windows NT 기계에서 AFS 서버 프로세스를 실행합니다. Windows NT에 있는 AFS 서버는 AFS 파일 서버, AFS 데이터베이스 서버, AFS 백업 서버 및 AFS 시스템 제어 서버 중 하나 이상으로 구성될 수 있습니다.

AFS 서버 설치 요구조건

사용자 시스템은 Windows용 AFS 서버 구성요소를 설치하기 위해 다음과 같은 하드웨어, 소프트웨어 및 관리 특권 요구조건을 충족시켜야 합니다.

하드웨어

AFS 서버 설치를 위한 하드웨어 요구조건은 다음과 같습니다.

  • 80486/66MHz(이상) 마이크로프로세서 또는 펜티엄 기반 마이크로프로세서를 사용하는 PC

    권장사항: 펜티엄이나 그 이상의 마이크로프로세서를 사용하는 PC

  • 최소 16MB의 RAM

    권장사항: 32MB 이상의 RAM

  • 최소 20MB의 사용가능한 디스크 공간을 가진 하드 디스크
  • AFS 파일 시스템 데이터를 보유하기 위한 최소 하나 이상의 NTFS 디스크 볼륨. NTFS 볼륨은 AFS 파티션으로서 구성하는 데 적합한 Windows NT Recycle Bin 이외의 데이터나 파일을 포함하지 말아야 합니다.

소프트웨어

AFS 서버를 설치를 위한 소프트웨어 요구조건은 다음과 같습니다.

  • 서비스 팩 4 또는 5가 있는 Microsoft Windows NT 4.0

관리 특권 요구조건

AFS 서버를 설치, 구성 및 시작하려면 Windows 시스템에서 로컬 관리자 그룹의 구성원이어야 합니다.

제품 주의사항

이 절은 Windows용 AFS 버전 3.6의 AFS 서버 구성요소에 대한 중요하고도 일반적인 정보를 강조합니다. 이 절에는 이 AFS 서버 릴리스에서 사용가능한 새 기능 및 확장사항에 대한 설명이 들어 있습니다.

2 GB보다 큰 볼륨에 대한 지원

AFS 볼륨은 쉽게 관리되는 단일 장치로 구성되는 관련 파일의 모음이며 네트워크에서 로드 밸런스를 용이하게 하기 위해 파일 서버 기계 간에 쉽게 이동될 수 있습니다. 각 AFS 볼륨은 연관된 고유의 할당량을 갖습니다. 볼륨의 할당량은 볼륨에 있는 정보가 차지할 수 있는 최대 디스크 공간의 양을 지정합니다. 할당량은 볼륨 기준으로 설정되므로, 더 많은 자료를 포함하는 볼륨에 대해서는 증가되고, 추가 공간이 필요하지 않은 볼륨에 대해서는 감소될 수 있습니다. Windows용 AFS, 버전 3.6은 이제 2 GB보다 큰 할당량을 가진 볼륨을 지원합니다.

시스템 제어 서버가 사용하는 암호화 루틴에는 Windows용 AFS의 모든 버전이 포함됩니다

시스템 제어 서버 기계에서 갱신 서비스가 사용하는 암호화 루틴은 이제 미국 정부 수출 규제의 완화로 인해 모든 Windows용 AFS 버전에 포함되고 있습니다. 이전에는 정부 규제로 이 암호화 루틴의 수출이 금지되었습니다. 그러므로, AFS의 미국 배포가 실행되지 않은 사이트는, 허용된 감지 시스템 정보가 암호화되지 않은 상태에서 전송되므로, 시스템 제어 서버를 사용하여 구성 파일을 분배할 수 없었습니다. 이제, 모든 사이트는 시스템 제어 서버 기계를 사용하여 AFS 서버 구성 정보의 새 버전을 모든 AFS 파일 서버 기계로 분배할 수 있습니다.

이름 서비스 호스트 이름에 해당하는 AFS 서버 컴퓨터 이름

AFS 서버 기계의 컴퓨터 이름(네트워크 대화 상자의 식별 탭에서의 컴퓨터 이름 필드에 표시됨)은 인터넷 프로토콜(IP) 주소로 이름을 맵하기 위해 사용되는 이름 서비스(보통 도메인 이름 서비스 또는 DNS)에 의해 컴퓨터에 지정된 호스트 이름과 일치해야 합니다. 예를 들어, AFS 서버가 설치된 기계가 컴퓨터 이름 afsserver1을 가지고 있으며 yourcompany.com 도메인에 있는 경우, 기계의 해당 DNS 입력 항목은 afsserver1.yourcompany.com이어야 합니다.

시각(Time Clock) 동기화

보안 및 데이터베이스 복제 프로토콜을 지원하려면, AFS 서버를 실행하는 모든 기계에 있는 시각은 동기화되어야 합니다. 시간 동기화 소프트웨어는 많은 소스에서 얻을 수 있습니다. 예를 들어, Microsoft는 Windows NT Server Resource Kit으로 timeserv 시간 동기화 서비스를 분배합니다.

AFS 서버 기계 보안

AFS 서버 소프트웨어가 설치된 기계가 보안되지 않으면(즉, 비관리 요원이 기계에 로그인할 수 있는 경우), 기계에 있는 모든 AFS 파티션 및 AFS 서버 소프트웨어 디렉토리가 환경 및 파일 시스템 유형에 적합하게 보안되도록 강력히 권장합니다. 그러나, AFS 서버 기계를 보안할 때, Windows NT 국지 SYSTEM 핵심부는 AFS 소프트웨어 및 모든 AFS 파티션에 대한 전체 액세스를 가지며, 그렇지 않은 경우 AFS 프로세스는 실행할 수 없습니다.

AFS 파일 서버 기계 종료

AFS 파일 서버로서 실행 중인 Windows NT 기계를 종료하기 전에, 항상 기계에서 AFS 파일 서버를 정지하십시오.

AFS 파일 서버가 기계에서 시작되면, 프로세스는 구조 파일을 작성합니다. 파일 서버가 수동으로 정지되면, 이 구조 파일은 제거됩니다. 그러나, AFS 파일 서버가 실행 중인 동안 Windows NT 기계가 종료되면, 이 구조 파일은 제거되지 않습니다. 기계가 차후에 재시작되면, 파일 서버는 자동으로 시작하며, 구조 파일의 존재를 감지하는 순간에 구조 프로그램을 실행합니다. 구조 프로그램을 실행하는 것은 시간이 많이 소모되므로, 불필요하게 구조 프로그램 프로세스를 실행하는 것을 피하기 위해 기계를 종료하기 전에 기계에서 AFS 파일 서버를 정지하는 것이 좋습니다.

Windows NT 기계에서 AFS 파일 서버를 종료하려면 다음 절차 중 하나를 사용하십시오.

  • Windows NT 제어판에서,
    1. 서비스 아이콘을 두 번 누르십시오. 서비스 대화 상자가 열립니다.
    2. 서비스 목록에서, IBM AFS 서버 서비스를 선택하십시오.
    3. 정지 단추를 선택하십시오. 서비스 정지를 확인하는 메시지 상자가 나타납니다. 예 단추를 선택하십시오.

      기계에 있는 AFS 파일 서버가 정지됩니다.

  • Windows NT 명령 프롬프트에서,
    1. 다음 명령을 입력하십시오. 
      bos shutdown -server machine_name -wait

      여기서 machine_name은 AFS 파일 서버가 실행 중인 Windows NT 기계의 이름입니다.

    2. 명령을 실행하려면 <Enter>를 누르십시오.
주:일반적으로, AFS 파일 서버로서 실행 중인 기계는 예방적인 유지보수를 수행하기 위해서만 종료됩니다.

서버 문제점 이벤트에서 크래쉬 덤프 파일 얻기

서버 장애 이벤트에서 크래쉬 덤프 파일을 작성하려면 모든 AFS 서버 기계를 구성하도록 권장합니다. 이 정보는 기술 지원 요원이 서버의 문제점을 진단하는 데 중요합니다. 응용프로그램 오류 발생 시 자동으로 2진 크래쉬 덤프 파일을 생성하는 Dr. Watson 유틸리티 구성 방법에 대한 정보는 Microsoft Windows NT 문서를 참조하십시오.

AFS 서버는 경로가 ANSI 문자만을 포함하는 디렉토리에 설치되어야 합니다

Windows용 AFS 소프트웨어는 경로에 ANSI 문자만이 들어 있는 디렉토리에만 설치될 수 있습니다(예를 들어, C:\Program Files).

임시 파일

Windows용 AFS 서버는 경우에 따라 임시 파일을 작성합니다. 파일이 작성되는 곳을 제어하려면, 환경 변수(TMP 또는 TEMP)를 선호하는 임시 디렉토리의 전체 경로에 설정하십시오. 임시 디렉토리를 지정하지 않는 경우, 임시 파일은 파일을 작성한 프로세스의 현재 작업 디렉토리에 작성됩니다.

AFS 서버에 대한 자국어 지원

AFS는 양방향 스크립트(Hebrew, Arabic 등)에 대한 지원과 함께, AFS 서버 그래픽 사용자 인터페이스(GUI) 도구 및 문서에 대한 자국어 지원을 제공합니다. 설치된 언어 문자열은 제어판의 국가별 설정 상자 대화 상자에 지정한 대로 사용자 기계의 시스템 기본 로케일에 의해 결정됩니다. 현재 로케일에 대한 언어 문자열이 없는 경우, 영어 문자열이 기본값으로 설치됩니다.
주:기계의 시스템 기본 로케일은 기계에서의 사용자 로케일(사용자가 지정)과 다를 수 있습니다. 그러나, Windows용 AFS가 설치된 언어 문자열은 항상 시스템 기본 로케일에 의해 결정됩니다.

kpwvalid Password Strength Checker는 Windows 시스템에서 사용되지 않습니다

AFS 사용자 암호가 Windows 시스템에서 변경되면, kpwvalid 프로그램은 새 암호를 확인하기 위해 사용되지 않습니다 (UNIX 시스템에서, kpwvalid 프로그램이 kpasswd 프로그램과 동일한 디렉토리에 존재하는 경우, kpwvalid 프로그램은 모든 새 사용자 암호를 확인합니다).

3.5 이전 데이터베이스 서버가 있는 셀로 서버를 구성하는 것은 계정 암호를 요구합니다

데이터베이스 서버가 Windows용 AFS 서버, 버전 3.6을 3.5 버전 이전의 AFS 버전을 실행 중인 AFS 셀로 구성하는 경우, 구성 프로세스 중 대화 상자는 AFS 계정 암호를 제공하도록 프롬프트합니다.

주:어떠한 이유로 AFS 계정 암호를 모르는 경우, 대신 AFS 계정 키를 나타내는 8진 숫자의 24문자열을 입력할 수 있습니다. 8진 키를 얻으려면, kas examine 명령이나 boslistkeys 명령을 AFS 데이터베이스 서버 기계에서 발행하십시오. AFS 계정 8진 키를 표시하려면 noauth 모드에서 서버를 실행중이어야 합니다.

제한사항 및 제약사항

이 절은 Windows용 AFS, 버전 3.6의 AFS 서버 구성요소에서 알려진 제한사항 및 제약사항을 설명합니다.

AFS bosserver 프로세스는 프로세스가 종료할 때 지정자 프로그램을 실행하지 않습니다

AFS bosserver 프로세스는 제어 하에 있는 프로세스가 종료할 때 등록된 지정자 명령(지정한 경우)을 현재 실행하지 않습니다.
