IBM AFS for Windows インストールの手引き

概要

AFS(R) は、 共用ファイル・スペースによって整合性のとれたファイル・アクセスを可能とするエンタープライズ・ファイル・システムです。 いくつかのファイル・サーバー のローカル・ファイル・システムを結合することにより、AFS はマシンの境界を意識させない 単一のファイル・スペースを実現します。 ファイルは、コンピューター・ネットワーク内のいろいろなマシンに保管されていますが、 エンタープライズ内のすべてのマシンからアクセス可能です。

IBM AFS for Windows(R)、バージョン 3.6 では、 AFS の完全な機能を Microsoft(R) Windows オペレーティング・システムで使用できるようになっています。

本書の概要

本書では、インストールの前提条件を要約し、AFS for Windows のインストール、構成、アンインストールの方法を詳述し、 インストールまたはアンインストールのプロセスでシステムに加えられる変更を概説します。 また、AFS for Windows に付属する文書についても説明します。

対象読者

本書に記載されている情報は、AFS for Windows に含まれる製品のインストールと構成を担当するシステム管理者やユーザーを対象としています。 本書では、システム管理者が一般的なシステム管理に通じており、 ユーザーが Microsoft Windows オペレーティング・システムの基本的な用語と概念に通じていることを想定しています。

AFS for Windows のインストール

このセクションでは、インストール可能な AFS コンポーネントの組み合わせを概説し、AFS for Windows のインストール手順を説明し、 インストール・プロセスによってシステムに加えられる変更点を示します。

AFS for Windows のコンポーネント

AFS for Windows、バージョン 3.6 には以下のコンポーネントが含まれます。

  • AFS サーバー

    AFS サーバーは、AFS サーバー・プロセスを実行します。また、AFS サーバーには、 セットアップを簡単にする AFS サーバー構成ウィザードが含まれています。

  • AFS コントロール・センター

    AFS コントロール・センターには、AFS システム管理者が AFS セル管理を行うのを助ける 2 つの強力なグラフィカル・ユーザー・インターフェース (GUI) ツール (AFS サーバー・マネージャーと AFS アカウント・マネージャー) が含まれています。

  • AFS クライアント

    AFS クライアントは、AFS ファイル・スペースへの直接アクセスを提供し、 ユーザーが AFS 内のファイルやディレクトリーを管理できるようにします。

  • AFS Light

    AFS Lightは、AFS Light ゲートウェイ・マシンを介して AFS ファイル・スペースへの直接アクセスを提供し、 ユーザーが AFS 内のファイルやディレクトリーを管理できるようにします。

  • AFS 補足資料

    AFS 補足資料は、AFS システム管理に関する追加情報を提供します。 これには、IBM AFS 管理の手引き および IBM AFS 管理解説書 が含まれます。

インストール・オプション

ご使用の Windows オペレーティング・システムに基づき、 さまざまな組み合わせの AFS for Windows コンポーネントをインストールできます。 それぞれの AFS for Windows コンポーネントごとの特定のソフトウェア要件についての詳細は、 IBM AFS for Windows リリース情報 を参照してください。 AFS サーバーをインストールする場合は、AFS クライアント、 バージョン 3.6 がマシンにインストールされていなければ、 AFS クライアントもインストールする必要があります。 インストールするコンポーネントにかかわらず、AFS for Windows をインストールするにはで説明されているインストール手順に従ってください。
注: AFS for Windows のセットアップ・プログラムを変更して、 クライアント・コンポーネントを除くすべてのコンポーネントを使用禁止にするオプションがあります。 このようなクライアントのみのセットアップ・プログラムによって、 ユーザーは AFS クライアント以外のコンポーネントをインストールすることができなくなります。 クライアントのみのインストールを実行するには、 他のインストール・ファイルと同じディレクトリーにファイル setup.co を作成してください。 これによって、セットアップ・プログラムは AFS クライアントのみがインストールされるようにします。 setup.co ファイルの内容は無関係です。 実行するインストールのタイプにかかわらず、 AFS for Windows をインストールするにはで説明されているインストール手順に従ってください。

旧バージョンからのアップグレード

Windows NT マシンでは、 ソフトウェアのアップグレードを行う目的で AFS for Windows のコンポーネントをアンインストールする必要はありません。 本リリースの AFS for Windows をシステムにインストールする際に、 既存のソフトウェアの除去や構成解除を行う必要はありません。 AFS for Windows をアップグレードするには、 AFS for Windows をインストールするにはで説明されているインストール手順に従ってください。 インストール・プロセスでは、旧バージョンの AFS コンポーネントがアップグレードされ、AFS 構成情報は保持されます。

Windows 95 または Windows 98 マシンでは、AFS Light コンポーネントのアップグレードを行う前に、 AFS for Windows をアンインストールするにはの説明に従って、 以前にインストールした AFS Light をアンインストールする必要があります。

AFS for Windows のインストール・ツールでは、 異なるバージョン番号の AFS コンポーネントをインストールすることはできません。 また、マシン上に複数の AFS for Windows コンポーネントがある場合、 他のすべてのコンポーネントをアップデートせずに 1 つのコンポーネントだけをアップデートすることもできません。

AFS for Windows をインストールするには

AFS for Windows をインストールする前に、 IBM AFS for Windows リリース情報 で、 インストールの前提条件の詳細を参照してください。 他の Windows アプリケーションを実行している場合は、AFS for Windows をインストールする前に終了することをお勧めします。

  1. AFS for Windows インストール・ディスクを CD-ROM ドライブに挿入します。
  2. 以下のいずれかの方法により、AFS for Windows セットアップ・プログラムを実行します。
    • [スタート] メニューから、 [ファイル名を指定して実行] を選択します。 drive:¥setup と入力します。 ここで、drive は CD-ROM ドライブのドライブ名です。 [OK] をクリックします。
    • Windows エクスプローラで CD-ROM ドライブを選択し、 setup.exe プログラムをダブルクリックします。
  3. [ようこそ] ダイアログ・ボックスが表示されます。 [次へ] をクリックしてインストール・プロセスを続行します。
  4. [コンポーネントの選択] ダイアログ・ボックスが表示されます。
    • [コンポーネント] ボックスで、 インストールまたはアップグレードする AFS for Windows コンポーネントを選択します。 Windows マシンにインストールできるコンポーネントのさまざまな組み合わせについては、 インストール・オプションを参照してください。 AFS サーバーをインストールする場合は、 AFS クライアント、バージョン 3.6 がシステムにインストールされていなければ、 AFS クライアントもインストール (またはアップグレード) 対象として選択する必要があります。
    • [Destination Folder (宛先フォルダー)] ボックスは、 選択したコンポーネントがインストールされるデフォルトのドライブとディレクトリーを示します。 デフォルト・ドライブは Windows がインストールされているドライブです。 デフォルト・ディレクトリーは、 そのドライブの ¥Program Files¥Ibm¥Afs です。 別のドライブやディレクトリーを選択するには、 [参照] ボタンを選択します。
      注:旧バージョンの AFS for Windows からアップグレードする場合や、 AFS for Windows を再インストールする場合は、 以前にインストールされていたバージョンと同じインストール・ディレクトリーを選択する必要があります。

    [次へ] をクリックしてインストール・プロセスを続行します。

  5. 選択した AFS for Windows コンポーネントのアプリケーション・ファイルがシステムにインストールされます。 インストール・プロセスが完了すると、 [セットアップ完了] ダイアログ・ボックスが表示され、 インストールした AFS 製品を使用する前にシステムを再始動する必要があることを示します。 [はい、すぐに再起動します] を選択してから、 [完了] ボタンを選択してください。 システムはシャットダウンののち再始動します。

    AFS for Windows のインストールは完了しました。

Windows NT 版 AFS クライアントをインストールすると、 次のような変更がシステムに加えられます。

  • [スタート] メニューに、 以下のプログラム・アプリケーションを含むプログラム・グループ [IBM AFS] が作成されます。

    [Documentation (文書)] プログラム・エントリーでは、 AFS for Windows に付属する一連の AFS オンライン文書にアクセスできます。

    [クライアント] プログラム・サブグループでは、AFS クライアントのプロパティー・ボックスと AFS クライアントのオンライン・ヘルプにアクセスできます。

  • afsdcell.ini ファイルが Windows ディレクトリーと ¥Program Files¥Ibm¥Afs¥Common ディレクトリーに入れられます。 旧バージョンの AFS クライアントからアップグレードした場合、 Windows ディレクトリーにある AFS クライアントのセル・データベース (afsdcell.ini) は置き換えられません。

AFS for Windows の構成

このセクションでは、AFS for Windows の各コンポーネントを構成する手順について詳述します。 AFS を使用する前に、システム上のコンポーネントを構成する必要があります。

AFS クライアントを構成するには

注:Windows NT システム上で AFS サーバーを構成することになっている場合、 AFS クライアントを構成する必要はありません。 AFS クライアントは AFS サーバーの構成時に自動的に構成されます。 また、旧バージョンの AFS クライアントからこのバージョンの AFS for Windows にアップグレードした場合は、 構成情報が保存されます。 AFS クライアントを再構成する必要はありません。
  1. [スタート] メニューから [設定] を選択し、 次いで [コントロール パネル] を選択します。
  2. [AFS クライアント構成] アイコンをダブルクリックします。 AFS クライアント構成ユーティリティーがオープンし、 [一般] タブが表示されます。
  3. [セル名] ボックスに、 このマシンがクライアントとなる AFS セルの名前を入力します。
  4. [AFS セル] タブを選択します。 このマシンがクライアントとなるセルが AFS セルのリストに表示されていない場合は、 [追加] ボタンを選択します。 [新規セル] ダイアログ・ボックスがオープンします。 [AFS セル] ボックスにセル名を入力し、 [記述] ボックスに短い説明を入力してください。

    [追加] ボタンを選択します。 [サーバーの追加] ダイアログ・ボックスがオープンします。 [サーバー名] ボックスに、 選択したセルにあるボリューム・ロケーション・サーバーの名前を入力してください。 [了解] を選択すると、[サーバーの追加] ダイアログ・ボックスがクローズします。 このプロセスを繰り返して、 セルにあるすべてのボリューム・ロケーション・サーバーの情報を追加してください。 (AFS セルにあるボリューム・ロケーション・サーバーの名前が分からない場合は、AFS システム管理者に尋ねてください。) すべてのサーバー情報を入力し終えたら、 [了解] を選択して [新規セル] ダイアログ・ボックスをクローズします。

  5. [一般] タブを選択してから、 [サービス開始] ボタンを選択して、 AFS クライアント・サービスを開始します。
  6. [ドライブ名] タブを選択します。 Windows NT マシン上のドライブ名を AFS ファイル・スペースにマップするには、 [追加] ボタンを選択します。 [ドライブ名のマッピング] ダイアログ・ボックスがオープンします。
  7. [ドライブ名] ボックスで、 AFS ファイル・スペースにマップするドライブを選択するか、デフォルトを受け入れます。 [AFS パス] ボックスでは、 選択したドライブをマップする AFS ロケーション (たとえば ¥afs) を指定します。 必要であれば、AFS ドライブ・マッピングの説明を [記述] ボックスに入力します。 [了解] を選択すると、AFS ファイル・スペース内の指定位置にドライブが接続されます。
  8. [了解] を選択して、AFS クライアント構成ユーティリティーをクローズします。

    選択した AFS セルで AFS クライアントが構成されました。 選択したドライブ・マッピングにより Windows NT エクスプローラーで AFS ファイル・スペースにアクセスすることができます。

AFS クライアントを AFS Light ゲートウェイとして構成するには

Windows NT マシン上の AFS クライアントを、 AFS Light ゲートウェイとして機能するように構成することができます。 AFS クライアントを AFS Light ゲートウェイとして構成すると、 AFS Light ユーザーが AFS ファイル・スペースにアクセスできるようになります。

  1. AFS クライアントを構成するにはの説明に従って AFS クライアントを構成します。
  2. [スタート] メニューから [設定] を選択し、 次いで [コントロール パネル] を選択します。
  3. [AFS クライアント構成] アイコンをダブルクリックします。 AFS クライアント構成ユーティリティーがオープンし、 [一般] タブが表示されます。
  4. [AFS Light にゲートウェイを提供] オプションを選択します。
  5. [了解] を選択します。
    • AFS クライアント・サービスが実行されている場合は、 メッセージ・ボックスが表示され、サービスを再始動する必要があることを知らせます。 [はい] を選択すると、 AFS クライアント・サービスが再始動され、AFS Light ゲートウェイが使用可能になります。
    • AFS クライアント・サービスが停止している場合は、 メッセージ・ボックスが表示され、サービスを開始する必要があることを知らせます。 [はい] を選択すると、 AFS クライアント・サービスが開始され、AFS Light ゲートウェイが使用可能になります。
  6. セル・エントリーを AFS Light ゲートウェイのセル・データベースに追加してください。 AFS Light ユーザーがセルにアクセスするためには、 そのセルのエントリーが AFS Light のセル・データベースと AFS Light ゲートウェイのセル・データベースの両方に存在している必要があります。 ゲートウェイ・マシン内のセルの情報が間違っている、または欠落していると、 light クライアント・マシンはファイルにアクセスできなくなります。

    セル・データベースにエントリーを追加するには、以下のようにします。

    AFS Light 構成ユーティリティーから [AFS セル] タブにアクセスして、 [追加] ボタンを選択します。 [New Cell (新しいセル)] ダイアログ・ボックスがオープンします。 [AFS セル] ボックスにセル名を入力し、 [記述] ボックスに短い説明を入力してください。

    [追加] ボタンを選択します。 [サーバーの追加] ダイアログ・ボックスがオープンします。 [サーバー名] ボックスに、 選択したセルにあるボリューム・ロケーション・サーバーの名前を入力してください。 [了解] を選択すると、[サーバーの追加] ダイアログ・ボックスがクローズします。 このプロセスを繰り返して、 セルにあるすべてのボリューム・ロケーション・サーバーの情報を追加してください。 (AFS セルにあるボリューム・ロケーション・サーバーの名前が分からない場合は、AFS システム管理者に尋ねてください。) すべてのサーバー情報を入力し終えたら、 [了解] を選択して [新規セル] ダイアログ・ボックスをクローズします。

これで、Windows NT マシンは AFS Light ゲートウェイとして構成されました。 AFS Light ゲートウェイとして構成された AFS クライアント・マシンは、 Windows のコンテキスト内の AFS Light ユーザーを認証できなければなりません。 この認証は、ドメイン ・ユーザー・アカウント経由か、 または同期の マシン ・ユーザー・アカウント経由で行われます。 ドメイン・ユーザー・アカウントは、Windows ドメイン内のユーザー・アカウントです。 マシン・ユーザー・アカウントは、特定のホスト・マシン上でのみ有効なユーザー・アカウントです。

AFS Light ゲートウェイが Windows ドメイン内に構成されている場合、 AFS Light ユーザーは、そのゲートウェイが属するドメイン内のドメイン・ユーザー・アカウントか、 または、ゲートウェイ・ドメイン内のドメイン・ユーザー・アカウントと同じユーザー名およびパスワードを持つマシン・ユーザー・アカウントにログオンしなければなりません。

マシン・ユーザー・アカウントが使用されている場合、それらのアカウントは、 AFS Light ゲートウェイおよび AFS Light マシン上で同期している必要があります。 ユーザーは、AFS Light ゲートウェイ・マシン上で定義されているマシン・ユーザー・アカウントと同じユーザー名およびパスワードの AFS Light マシンにログオンしなければなりません。

AFS Light を構成するには

AFS Light は AFS Light ゲートウェイを介して AFS ファイル・スペースにアクセスします。 AFS Light を構成するには、 Windows NT マシンで AFS クライアントが実行されており、 AFS Light ゲートウェイとして構成されている必要があります。 詳細については、AFS クライアントを AFS Light ゲートウェイとして構成するにはを参照してください。

  1. [スタート] メニューから [設定] を選択し、 次いで [コントロール パネル] を選択します。
  2. [AFS Light 構成] アイコンをダブルクリックします。 AFS Light 構成ユーティリティーがオープンし、 [一般] タブが表示されます。
  3. [ゲートウェイ] ボックスに、 AFS Light ゲートウェイとして構成されている Windows NT マシンの名前を入力して、 [Connect Now (即時接続)] をクリックします。 ゲートウェイ・マシンの名前は、 mach-afs という形式のゲートウェイの NetBIOS サービス名です。 ここで、mach は、ホスト・コンピューターの名前が最大 11 文字まで入ります。 AFS Light は、ゲートウェイ・マシンと通信するためには、 このサービス名を解決できなければなりません。 名前を解決するには、ゲートウェイの NetBIOS サービス名をクライアントの LMHOSTS ファイルか、 あるいは適切な DNS または WINS サーバーに追加します。 AFS Light マシンおよびその AFS Light ゲートウェイ・マシンが同一のサブネットに常駐している場合は、 構成を追加する必要はありません。

    AFS Light は、自動的に AFS Light ゲートウェイと同じセルのメンバーになります。 セルの名前は [セル名] ボックスに表示されます。

    注:AFS Light ゲートウェイ・マシンが AFS Light マシンと同一のドメイン内にあり、 このドメインにあるゲートウェイ・マシンのホスト名が xyz-pc である場合は、 [ゲートウェイ] ボックスにはコンピューター名として xyz-pc または xyz-pc.xcompany.com を指定することができます。

  4. [AFS セル] タブを選択します。 このマシンが属しているセルが AFS セルのリストに表示されていない場合は、 [追加] ボタンを選択します。 [New Cell (新しいセル)] ダイアログ・ボックスがオープンします。 [AFS セル] ボックスにセル名を入力し、 [記述] ボックスに短い説明を入力してください。

    [追加] ボタンを選択します。 [サーバーの追加] ダイアログ・ボックスがオープンします。 [サーバー名] ボックスに、 選択したセルにあるボリューム・ロケーション・サーバーの名前を入力してください。 [了解] を選択すると、[サーバーの追加] ダイアログ・ボックスがクローズします。 このプロセスを繰り返して、 セルにあるすべてのボリューム・ロケーション・サーバーの情報を追加してください。 (AFS セルにあるボリューム・ロケーション・サーバーの名前が分からない場合は、AFS システム管理者に尋ねてください。) すべてのサーバー情報を入力し終えたら、 [了解] を選択して [新規セル] ダイアログ・ボックスをクローズします。

    AFS Light ゲートウェイのセル・データベース (afsdcell.ini file) 内には、 AFS Light ユーザーがセルを認証するための同一のエントリーが存在している必要があります。 ゲートウェイ・マシンのセル・データベースと light クライアントのセル・データベースの同期化について、 詳しくは AFS クライアントを AFS Light ゲートウェイとして構成するには を参照してください。

  5. [ドライブ名] タブを選択します。 Windows NT マシン上のドライブ名を AFS ファイル・スペースにマップするには、 [追加] ボタンを選択します。 [ドライブ名のマッピング] ダイアログ・ボックスがオープンします。
  6. [ドライブ名] ボックスで、 AFS ファイル・スペースにマップするドライブを選択するか、デフォルトを受け入れます。 [AFS パス] ボックスでは、 選択したドライブをマップする AFS ロケーション (たとえば ¥afs) を指定します。 必要であれば、AFS ドライブ・マッピングの説明を [記述] ボックスに入力します。 [了解] を選択すると、AFS ファイル・スペース内の指定位置にドライブが接続されます。
  7. [了解] を選択して、AFS Light 構成ユーティリティーをクローズします。

    指定した AFS セルで AFS Light が構成されました。 ドライブ・マッピングにより Windows エクスプローラーで AFS ファイル・スペースにアクセスすることができます。

AFS サーバーを構成するには

構成プロセスでは、 AFS サーバーの実行に必要なサービスを開始し、Windows NT マシン上で AFS パーティションをセットアップします。 AFS 構成ウィザードを使用すれば、 AFS サーバーを新しい AFS セル内の最初のサーバーとして、 または既存の AFS セル内のサーバーとして、 手軽に構成することができます。 このバージョンの AFS サーバーにアップグレードされている場合、 旧バージョンの構成情報は保存されます。サーバーを再構成する必要はありません。

AFS サーバーをセル内の最初の AFS サーバーとして構成するには

  1. [スタート] メニューから [プログラム] を選択し、 [IBM AFS]->[サーバー]->[構成ウィザード] を選択します。 AFS サーバー・クィック・スタート・ウィザードがオープンします。
  2. [次へ] ボタンを選択します。 [セルおよびサーバー情報] ダイアログ・ボックスが表示されます。
  3. [新規 AFS セルの最初のサーバーにする] オプションを選択します。
  4. [セル名] ボックスに、新しい AFS セルの名前を入力します。

    AFS セルの名前として使用可能なインターネット・ドメイン名の形式については、 以下の制約事項が適用されます。

    • セル名は、セルを AFS グローバル・ネーム・スペースにあるほかのすべての名前から区別できるよう、 固有の名前でなければなりません。
    • セル名には、最大で 64 文字まで使用することができます。 ただし、短い名前を使用するようお勧めします。
    • 異なるタイプのオペレーティング・システムで使用できることを保証するために、 セル名には、小文字、数字、下線、ダッシュ、およびピリオドだけを使用する必要があります。
    • セル名は、任意の英数字を含むことができ、ピリオドによって区切られるという規則があります。
    • セル名は、国をあらわす .jp で終わるようにします。 その直前は、セルが属している機関のタイプを示すサフィックスになります。 標準的なサフィックスとしては、 .co (ビジネスおよび他の商業組織)、 .ne (インターネット・プロバイダー)、 .ac (大学などの教育機関)、 .go (政府機関) などがあります。
  5. [パスワード] ボックスに、 セル内の AFS サーバー・プリンシパル・アカウント (afs) のパスワードとなる文字列を入力します。 すべての AFS サーバーはこのプリンシパルとして AFS トークンを取得します。 また、認証サーバーのチケット許可サービス (TGS) モジュールは、 AFS クライアントが相互認証の際にサーバーに提示するサーバー・チケットを、 このパスワードを使用して暗号化します。
  6. [パスワードの検査] ボックスに、 このセルの AFS サーバー・プリンシパル・アカウントが使用する最初の AFS パスワードを再入力して、 選択したパスワードを確認します。
  7. [次へ] ボタンを選択します。 [管理情報] ダイアログ・ボックスが表示されます。
  8. [名前] ボックスに、 このセルの汎用 AFS 管理アカウントとして機能するユーザー名 (一般には admin) を入力します。

    汎用の管理アカウントを使用すれば、 それぞれのシステム管理者に特権を付与する必要はありません。 その代わり、各管理者はこの汎用管理アカウントの名前とパスワードを知っており、 管理特権を必要とするタスクを実行する際にはこの ID を使用して AFS に認証します。

  9. [パスワード] ボックスに、 AFS 管理アカウントのパスワードとなる文字列を入力します。
  10. [パスワードの検査] ボックスに、 AFS 管理アカウントのパスワードを再入力して、選択したパスワードを確認します。
  11. AFS 管理アカウントに割り当てられる AFS ユーザー ID (UID) を指定します。
    • (推奨) 次に使用可能な UID を AFS 管理アカウントに自動的に割り当てるには、 [次に使用可能な AFS UID を使用] オプションを選択します。
    • 特定の UID を AFS 管理アカウントに割り当てるには、 [この AFS UID を使用] オプションを選択して、 その UID を入力ボックスに入力します。

      注:AFS UID を既存の UNIX UID と一致させる必要がある場合を除き、 特定の UID を新しい AFS アカウントに割り当てることは一般に勧められていません。

  12. [次へ] ボタンを選択します。 [File Service (ファイル・サービス)] ダイアログ・ボックスが表示されます。

    AFS ファイル・サーバーは、 要求されたファイルやデータをサーバーから AFS クライアントに送ります。 ファイル・サーバーは、ファイルやデータの保管、 ファイルやディレクトリーのコピー、移動、作成、および削除要求の処理、 サーバー上のファイルやディレクトリーに関する状況情報のトラックを行います。

    ここでは、新しいセル内の最初の AFS サーバーを構成しているので、 ファイル・サービスをサーバー上で構成する必要があります。 この構成は自動的に行われます。

  13. [次へ] ボタンを選択します。 [Database Service (データベース・サービス)] ダイアログ・ボックスが表示されます。

    各 AFS セルには 1 つ以上のデータベース・サーバーが含まれていなければなりません。 各データベース・サーバーは、 AFS データベース (認証データベース、 保護データベース、ボリューム・ロケーション・データベース、 および任意選択でバックアップ・データベース) を保守するデータベース・プロセスを実行します。

    ここでは、新しいセル内の最初の AFS サーバーを構成しているので、 データベース・サービスをサーバー上で構成する必要があります。 この構成は自動的に行われます。

  14. [次へ] ボタンを選択します。 [Backup Server (バックアップ・サーバー)] ダイアログ・ボックスが表示されます。

    バックアップ・サーバーは、 バックアップ・システムに関連した情報が保管されるバックアップ・データベースを保守します。 バックアップ・サーバーにより、AFS システム管理者は、 AFS ファイル・スペース内のデータをボリュームからテープにバックアップすることができます。 データがファイル・システムから逸失した場合 (たとえば、 システムまたはディスクの障害によってデータが逸失した場合) には、 そのデータをテープから復元することができます。

  15. この AFS サーバーをバックアップ・サーバーとして構成する場合は、 [はい、バックアップ・サーバーとして構成します] オプションを選択します。 この AFS サーバーをバックアップ・サーバーとして構成しない場合は、 [いいえ、バックアップ・サーバーとして構成しません] オプションを選択します。

    注:バックアップ・サーバーをセル内のいずれかのデータベース・サーバーで構成すると、 そのバックアップ・サーバーはセル内のすべての データベース・サーバーで構成されます。

  16. [次へ] ボタンを選択します。 [AFS パーティション] ダイアログ・ボックスが表示されます。

    各 AFS ファイル・サーバーでは、 少なくとも 1 つのパーティションが AFS ボリュームの保管専用として指定されていなければならず、 すべての AFS ボリュームは、AFS パーティションとして指定されたパーティションに存在していなければなりません。 Windows NT マシンでは、NTFS ボリュームのみを AFS パーティションとして指定できます。 加えて、AFS パーティションは、 空になっている (または、Windows NT のごみ箱のみを含んでいる) NTFS ボリュームでのみ作成できます。

    ここでは、新しいセル内の最初の AFS サーバーを構成しているので、 このサーバー上の AFS パーティションを指定する必要があります。

  17. NTFS ボリュームのリストから、 AFS パーティションとして指定するボリュームを選択します。 [AFS パーティション名] ボックスには、 パーティション名の最後の部分を入力します。

    注:AFS サーバー上には、最大で 256 個の AFS パーティションが存在できます。 規則により、 各パーティションには /vicepx という名前が付けられます。 ここで、x は 1 つまたは 2 つの英小文字です。 AFS パーティションには、 /vicepa、 /vicepb、 ... /vicepz という名前が付けられます。 追加パーティションには、 /vicepaa、 ... vicepaz、 ... /vicepiv という名前が付けられます。

    NTFS ボリュームのドライブ名をパーティション名の最後の文字として使用することを強くお勧めします。

  18. [次へ] ボタンを選択します。 [Root AFS Volumes (ルート AFS ボリューム)] ダイアログ・ボックスが表示されます。

    ルート AFS ボリュームは、 すべての AFS セルのファイル・システムに組み込まなければならない 2 つのボリュームであり、 次のような名前が付けられています。

    • AFS ファイル・スペースの最上位のレベル (/afs) に対応するボリュームには、root.afs
    • セルの名前で /afs のすぐ下に取り付けられたボリュームには、 root.cell (たとえば、yourcompany.com セルの /afs/yourcompany.com)

    ここでは、新しいセル内の最初の AFS サーバーを構成しているので、 セルのルート・ボリュームをサーバー上で構成する必要があります。 この構成は、サーバーの構成時に自動的に行われます。

  19. [次へ] ボタンを選択します。 [複製] ダイアログ・ボックスが表示されます。

    AFS の複製機能を活用するには、 AFS ルート・ボリュームを複製する必要があります。 複製プロセスでは、AFS ボリュームの読み取り専用コピーが 1 つ以上作成され、 それらのコピーが 1 つ以上のサイト (AFS パーティションおよびサーバー) に配布されます。 複製により、AFS ボリュームの内容に 1 つ以上の AFS ファイル・サーバー・マシンからアクセスできるようになり、 システムの効率とデータの使用可能性が向上します。

    ここでは、新しいセル内の最初の AFS サーバーを構成しているので、 セルのルート・ボリュームをサーバー上で複製する必要があります。 この複製は、サーバーの構成時に自動的に行われます。

  20. [次へ] ボタンを選択します。 [システム制御サービス] ダイアログ・ボックスが表示されます。

    国内バージョンの AFS for Windows を実行しているセルでは、 システム制御センターにより、 新バージョンの AFS サーバーの構成情報がすべての AFS サーバーに配布されます。 一般には、AFS セルの最初のサーバーをシステム制御サーバーとして指定することをお勧めします。 (国際バージョンの AFS for Windows を実行しているセルでは、 システム構成ファイルを配布するのにシステム制御サーバーを使用しません。)
    注:システム制御センターの役割は、 後で異なるサーバー・マシンに割り当てることもできます。 ただし、セル内の AFS サーバーの数によっては、 別のマシンへの役割の割り当てには非常に時間がかかる可能性があります。

  21. この AFS サーバーを AFS セルのシステム制御サーバーとして構成する場合は、 [システム制御サーバーとして構成する] オプションを選択します。 この AFS サーバーを AFS セルのシステム制御サーバーとして構成しない場合は、 [システム制御サーバーとして構成しない] オプションを選択します。
  22. [次へ] ボタンを選択します。[システムの構成] ダイアログ・ボックスが表示されます。

    この AFS サーバーの構成に使用するステップが表示されるので、 実際の構成プロセスを開始する前にステップを検討することができます。
    注:選択内容を検討または変更するために前のステップに戻るには、 [戻る] を選択します。

AFS for Windows のアンインストール

このセクションでは、アンインストールの前提条件を概説し、 AFS for Windows をアンインストールする方法を説明し、 アンインストール・プロセスによってシステムに加えられる変更点を示します。

再インストールおよびアップグレード

Windows NT マシンでは、 ソフトウェアの再インストールやアップグレードを行う目的で AFS for Windows のコンポーネントをアンインストールする必要はありません。 AFS for Windows を再インストールまたはアップグレードするには、AFS for Windows をインストールするにはで説明されているインストール手順に従ってください。 インストール・プロセスでは、 以前にインストールした AFS コンポーネントが置き換えられます。 AFS 構成情報は保持されます。

Windows 95 または Windows 98 マシンでは、AFS Light コンポーネントの再インストールまたはアップグレードを行う前に、AFS for Windows をアンインストールするにはの説明に従って、 以前にインストールした AFS Light をアンインストールする必要があります。

アンインストールの前提条件

AFS をアンインストールすると、すべての AFS アプリケーション・ファイルが削除されます。 他のアプリケーションで使用されていると、これらのファイルを削除することができません。 このため、AFS for Windows をアンインストールする前にすべての AFS ダイアログ・ボックスをクローズする必要があります。

マシンを取り外す目的で AFS サーバーをアンインストールする場合は、 データの損失を避けるために以下の前提条件を満たす必要があります。

  1. AFS サーバーがファイル・サーバーとして機能している場合は、 すべての読み取り / 書き込みボリュームを別の AFS ファイル・サーバーに移動し、 すべての読み取り専用ボリュームを除去します。
  2. AFS サーバーを構成解除します。 AFS サーバー構成ユーティリティーをオープンし、 [サーバー] タブを選択します。 すべてのチェック・ボックスをクリアしてから、 [OK] を選択してください。

AFS for Windows をアンインストールするには

  1. [スタート] メニューから [設定] を選択し、 次いで [コントロール パネル] を選択します。
  2. [アプリケーションの追加と削除] アイコンをダブルクリックします。 [アプリケーションの追加と削除] ダイアログ・ボックスが表示され、 [インストールと削除] タブが示されます。
  3. コントロール・パネルをクローズします。
  4. アンインストールする AFS コンポーネントを選択してから、 [追加と削除] ボタンを選択します。 [ファイルの削除の確認] ダイアログ・ボックスが表示され、 選択した AFS for Windows コンポーネントを削除してもよいかどうか尋ねてきます。 アンインストール手順を続行するには、[はい] をクリックします。
  5. AFS メッセージ・ボックスが表示され、 構成情報を保持するかどうかを尋ねてきます。 構成情報を保持する場合は [はい] を選択し、 すべての構成情報を削除する場合は [いいえ] を選択します。 (AFS 補足資料コンポーネントに関連した構成情報はありません。 このコンポーネントをシステムから除去する場合、 AFS メッセージ・ボックスは表示されません。)
  6. [コンピューターからプログラムを削除] ダイアログ・ボックスがオープンし、 システムから削除されるコンポーネントが表示されます。

    注:他のコンポーネントで必要とされない共用 AFS ファイルを除去するかどうかを尋ねるメッセージ・ボックスが表示されることもあります。 選択した AFS コンポーネントを完全に除去するには、 [すべて「はい」] をクリックします。

これで、選択した AFS for Windows コンポーネントはアンインストールされました。 複数の AFS for Windows コンポーネントの組み合わせをインストールした場合は、 ステップ 4 〜 6 を繰り返して各コンポーネントを別個に除去する必要があります。

システムに加えられる変更

AFS クライアントのアンインストール時にシステムに加えられる変更

AFS クライアントをアンインストールすると、 次のような変更がシステムに加えられます。

  • すべての AFS クライアント・ファイルが ¥Program Files¥Ibm¥Afs¥Client¥Program ディレクトリーから除去され、Client ディレクトリーが除去されます。 また、他の AFS コンポーネントがインストールされていなければ、 Ibm ディレクトリーも除去されます。
    注:AFS for Windows のセットアップ・プログラムによってインストールされたファイル以外のファイルが入っていると、 これらのディレクトリーは除去されません。

  • コントロール・パネルから [AFS クライアント構成] アイコンが除去されます。
  • Windows NT エクスプローラーのコンテキスト・メニューから [AFS] メニューが除去されます。
  • [IBM AFS クライアント] サービスが削除されます。
  • AFS クライアントに関係したレジストリー・エントリーがシステムから除去されます。 構成情報の保持を選択した場合、 一部の情報はアンインストール・プロセス後もレジストリーに残ります。
