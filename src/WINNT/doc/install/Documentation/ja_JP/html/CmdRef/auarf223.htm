管理解説書

pts removeuser

目的

保護データベース・グループからユーザーを削除します。

構文 

pts removeuser -user <user name>+  -group <group name>+
               [-cell <cell name>]  [-noauth]  [-force]  [-help]
   
pts rem -u <user name>+  -g <group name>+  [-c <cell name>]  
        [-n]  [-f]  [-h]

構文

pts removeuser コマンドは、 -user 引き数に指定されたユーザーまたはマシンを、 -group 引き数に指定されたグループから削除します。

ユーザーをグループに追加するには、 pts adduser コマンドを使用してください。グループ・メンバーシップをリスト表示する場合には、 pts membership コマンドを使用してください。グループからユーザーを削除し、そのグループの項目を単一ステップで完全に削除する場合には、 pts delete コマンドを使用してください。

注意

AFS は、各ユーザーのグループ・メンバーシップを認証時にコンパイルします。グループから削除されるときに有効なトークンを持っているユーザーは、そのトークンを破棄するか再認証されるまで、そのグループのメンバーに適用される特権を保持します。

オプション

-name
削除するユーザー項目の名前、またはマシン項目の IP アドレス (完全名、またはワイルドカードを使用した形式) を指定します。

-group
メンバーを削除するグループを指定します。

-cell
コマンドを実行するセルを指定します。詳細については、pts の概要に関する解説ページを参照してください。

-noauth
発行者に、非特権 ID anonymous を割り当てます。詳細については、 pts の概要に関する解説ページを参照してください。

-force
このフラグを使用すると、エラーや他の問題が発生したときに、最初のエラーでコマンドの実行を停止するのではなく、可能な限り継続します。

-help
このコマンドのオンライン・ヘルプを出力します。これ以外の有効なオプションはすべて無視されます。

例

次の例では、ユーザー smith を、グループ staff、および staff:finance から削除しています。最初の引き数 (ユーザー名) には単一のインスタンスしか指定できないため、スイッチの名前は不要です。 

   % pts removeuser smith staff staff:finance

次の例では、グループ bin-prot から、 ABC Corporation ネットワーク内のすべてのマシンを表す、 3 つのマシン項目を削除しています。 

   % pts removeuser -user 138.255.0.0 192.12.105.0 192.12.106.0 -group bin-prot

必要となる特権

-group 引き数に指定されたグループの、保護データベースの 5 番目の privacy フラグの設定によって、必要な特権が異なります (フラグを表示するには、 pts examine コマンドを使用します)。

  • ハイフンの場合、グループの所有者と、 system:administrators グループのメンバーだけが、メンバーを削除することができます。
  • 小文字の r の場合、グループのメンバーは他のメンバーを削除することもできます。

(5番目の フラグを大文字の R に設定することは、できません。)
