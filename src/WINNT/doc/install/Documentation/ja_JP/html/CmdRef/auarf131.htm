目的

fs コマンド・セットの概要

構文

fs コマンド・セットのコマンドは、AFS クライアント・マシンのキャッシュ・マネージャーとの主な管理インターフェースです。これは、クライアント・マシンで稼動中のアプリケーションに代わってファイル・サーバー・マシンから AFS データを取り出す役割を果たします。

fs コマンド・セットのコマンドには、いくつかのカテゴリーがあります。

  • キャッシュ・マネージャーがどのようにサーバー・マシンと対話を行うかを設定し、報告するコマンド: fs checkservers、fs getcellstatus、fs getserverprefs、 fs listcells、fs newcell、fs setcell、

    fs setserverprefs、fs sysname、および fs wscell

  • アクセス制御リスト (ACLs) を管理するためのコマンド: fs cleanacl、fs copyacl、fs listacl、および fs setacl
  • 特定のファイルまたはディレクトリーを格納するサーバー・マシン、ボリュームまたは区画を管理するコマンド: fs diskfree、 fs examine、fs listquota、fs quota、fs setquota、fs setvol、

    fs whereis、および fs whichcell

  • ローカル・クライアント・キャッシュおよびその関連情報を管理するコマンド: fs checkvolumes、fs flush、fs flushvolume、fs getcacheparms、および fs setcachesize
  • ボリューム・マウント・ポイントを管理するコマンド: fs lsmount、fs mkmount、および fs rmmount
  • モニターおよび追跡を制御するコマンド: fs debug および fs messages
  • キャッシュ・マネージャーと他のファイル・システムの対話を管理するコマンド: fs exportafs
  • ヘルプを呼び出すコマンド: fs apropos および fs help

キャッシュ・マネージャーおよび fs コマンドは、次の構成ファイルを使用し、保守します。

  • /usr/vice/etc/CellServDB ファイルは、管理者がマシンで作業中のユーザーのために AFS アクセスができるように、ローカル・セルとすべての外部セルのデータベース・サーバー・マシンをリストします。データベース・サーバー・マシンは、管理情報のデータベースを維持する認証、バックアップ、保護、およびボリューム・ロケーション (VL) サーバーの各プロセスを実行します。ユーザーがセルにアクセスするには、その root.cell ボリュームがローカル・ファイル・ツリーにマウントされていなければなりません。
  • /usr/vice/etc/ThisCell ファイルは、AFS データにアクセスする AFS コマンド・セットおよびキャッシュ・マネージャーに関するマシンのセル・メンバーシップを定義します。
  • /usr/vice/etc/cacheinfo ファイルは、キャッシュの構成パラメーターを定義します。これには、キャッシュのサイズや、メモリー内またはディスクにあるかどうかが含まれます。

またキャッシュ・マネージャーは、ファイル・サーバー・マシンから取り出したファイルを格納し、追跡するためのキャッシュ区画 (デフォルトでは /usr/vice/cache）に自動的にファイルを作成します。

詳細については、各ファイルの解説ページを参照してください。

オプション

次のフラグは、fs セットのどのコマンドでも使用できます。各コマンドの解説ページにも説明はありますが、ここでは詳しく説明します。

-help
標準出力ストリームにコマンドのオンライン・ヘルプ・メッセージを出力します。このフラグは、コマンドの他のオプションと一緒に使用してはなりません。このフラグを使用すると、コマンド・インタープリターは、他のすべての引き数およびフラグを無視し、ヘルプ・メッセージの出力だけを行います。

必要となる特権

fs コマンドに必要な特権は、他のコマンド・セットよりも多岐にわたっています。「必要となる特権」 セクションの各コマンドの説明には、特に注意を払ってください。

必要な特権には、次のようなものが含まれます。

  • ディレクトリーの ACL に対するアクセス権が必要です。たとえば、マウント・ポイントを作成したり除去するには、マウント・ポイントが常駐するディレクトリーの ACL で、 a (administer)、 i (insert)、および d (delete) のアクセス権が必要です。
  • ローカル・スーパーユーザー root としてマシンにログオンする。これは、キャッシュ・マネージャーの構成に影響するコマンドを発行するために必要です。
  • 保護データベースの system:administrators グループに属する。
  • 特権はありません。多数の fs コマンドが、ただ単に情報をリストします。
