管理解説書

fs sysname

目的

CPU/ オペレーティング・システムのタイプを報告または設定します。

構文 

fs sysname [-newsys <new sysname>]  [-help]
    
fs sy [-n <new sysname>]  [-h]

構文

fs sysname コマンドは、ローカル・マシンのカーネル・メモリーに記録されている CPU/オペレーティング・システム・タイプを設定または表示します。キャッシュ・マネージャーは、文字列を @sys 変数に置換します。これは AFS パス名で発生することがあります。@sys を使用した場合に、どのようにセルの構成が単純化されるかについては、AFS インストールの手引き および AFS 管理の手引き に説明があります。ただし、控えめに使用するのが最も良い方法です。それは、予測できないディレクトリーの変更をもたらすことがあるからです。

コマンドは、常にローカル・マシンのみに適用します。AFS に NFS/AFS Translator 経由でアクセスする NFS クライアント・マシンに発行した場合、文字列は NFS クライアント・マシンに設定または報告されます。AFS クライアント・マシン上で NFS クライアントの NFS/AFS 変換プログラムとして機能しているキャッシュ・マネージャーは、そのカーネル・メモリーに値を格納するので、ユーザーがバイナリー・ファイルへのパス名に @sys を含むコマンドを発行するとき、NFS クライアントに正しいバージョンのプログラムをバイナリー・ファイルで提供することができます。 NFS クライアント にログインした各ユーザーは個別に記録されます。これは、ユーザーがおそらくは UNIX の su コマンドを使用して NFS クライアントへのログイン・セッション中に、新規に ID (UNIX UID) を採用する場合、ユーザーは、正しい文字列が新規の IDに設定されているかどうか検査しなくてはならないことを示します。

オプション

-newsys
CPU/ オペレーティング・システムのインディケーター文字列をローカル・マシンに設定します。この引き数が省略された場合、出力が代わりに現在の設定を表示します。 AFS は標準化された文字列セットを使用します。AFS インストールの手引き または AFS Release Notes を参照してください。

-help
このコマンドのオンライン・ヘルプを出力します。これ以外の有効なオプションはすべて無視されます。

出力

-newsys 引き数が省略された場合、出力はマシンのシステム・タイプを次の形式で報告します。 

   Current sysname is 'system_type'

例

以下の例では、Solaris 5.7 を実行する Sun SPARCStation で作成された出力を示しています。 

   % fs sysname
   Current sysname is 'sun4x_57'

次のコマンドは、マシンが AIX 4.2 を実行する IBM RS/6000 であることを定義します。 

   % fs sysname -newsys rs_aix42

必要となる特権

現在の設定を表示するのに、特権は必要ありません。-newsys 引き数を AFS クライアント・マシンに使用するには、発行者がローカル・スーパーユーザー root としてログインしなくてはなりません。
