TE_device_name

目的

磁気テープ・コーディネーター・プロセスからのエラー・メッセージをログに記録します。

構文

TE_device_name ファイルには、device_name で示される磁気テープ装置またはバックアップ・データ・ファイルを制御するバックアップ・システムの磁気テープ・コーディネーター (butc プロセス) によって生成されるエラー・メッセージが記録されます。

磁気テープ・コーディネーターは、初期化時に、/usr/afs/backup ディレクトリーに ASCII 形式のファイルを作成します。ファイルが存在する場合は、磁気テープ・コーディネーターはその名前を TE_device_name.old に変更し、既存の TE_device_name.old ファイルがある場合は、それを上書きします。

磁気テープ装置の場合、磁気テープ・コーディネーターは、ローカルの /usr/afs/backup/tapeconfig ファイルで指定されている装置のパス名から最初の /dev/ という文字列を削除し、名前の中の残りのスラッシュを下線に置換して、ファイル名の可変部分の device_name を取り出します。たとえば、/dev/rmt/4m という装置のファイル名は TE_rmt_4m になります。同様に、バックアップ・データ・ファイルの場合、磁気テープ・コーディネーターは、最初のスラッシュ (/) を削除し、残りのスラッシュを下線で置換します。たとえば、 /var/tmp/FILE というバックアップ・データ・ファイルのファイル名は、TE_var_tmp_FILE になります。

ファイルのメッセージには、磁気テープ・コーディネーターが検出したエラーおよび警告条件が記述されます。たとえば、ダンプ操作時にアクセス不能になったボリュームや、磁気テープ・コーディネーターが磁気テープまたはバックアップ・データ・ファイルを上書きするときの警告が記述されます。このメッセージは、/usr/afs/backup/TL_device_name ファイルにも書き込まれます。このファイルには、磁気テープ・コーディネーターの大半のアクションが記録されます。
